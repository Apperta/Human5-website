Testing Strategies

Automated Testing

We are planing to use Mocha and Jasmine, two feature-rich JavaScript test frameworks, to ensure our JavaScript code run smoothly on the applications.
Mocha is one of the most popular JavaScript testing framework, it is running on both Node.js and in the browser for automatically testing to ensure the high quality of the code.
Jasmine is a BDD (behavior-driven development) framework which is widely used on testing JavaScript code. It can work on its own and does not rely any other frameworks.

Accessibility Testing

To make all users capable for using our applications, there are some situations we need to account for:

  • Different devices have different sreen sizes.
  • People who are blind.
  • People who are colour blind or have trouble on distinguishing low colour contrast.
  • People whose first language is not the language provided by the application want to use our app.

    • Accessibility Tools

    The main accessibility tool seems to be screen readers. A screen reader is a tool used to read what a sighted person will see on the page.
    Unlike on desktops you often need to use a paid for tool like JAWS, luckily on mobile you have free screen readers built-in the operating system.

  • For Android phones:
    • go to Settings > My Device > Accessibility > Vision (new to Android 5.0) > TalkBack > Turn TalkBack on.
  • For iOS phones:
    • go to Settings > General > Accessibility > VoiceOver > Trun VoiceOver on.

    Unit Testing

    Unit testing is a a process in which the smallest teatable parts of an application, called unites, are individually and independdently scrutinized for proper operation. This encourages developers to modify the source code without immediately concerns about how such changes might affect the functioning of other units or program as a whole.
