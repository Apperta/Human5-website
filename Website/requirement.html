Requirements

Our team decided together with our client to focus on the design and development of the application first and integrate user profiling later on.

MUST
  • Application must be easy-to-use allowing user to fill in data quickly
  • Must use Android and iOS as main platforms
  • ‘Knowledge base’ area, guiding users about the various aspects of the app
  • Have User Profile area with registration
  • Allow users to enter: TH5 ratings, Stress Level Ratings, WEMWBS
  • Allow users to create goals and thus set their deadlines
  • Have a built-in scoring algorithm to assess each set of inputs
  • Display user’s data graphically/diagrammatically to evaluate progress
  • Have a feature to store their data locally from any device
SHOULD
  • Send data periodically from local storage to database from the server
  • Access previously stored data from the local storage
  • Not limit user to enter data a specific number of times a day
  • The users can register through the app to a server.
COULD
  • Enable push notifications on both platforms
  • Allow users to form groups and thus set group goals
  • Have admin access feature where a selected superuser can access a particular user’s data and give them feedback
WONT
  • Desktop Version of the app

Use Cases

Use Case Save data
ID UC1
Brief Description System sends data to database
Primary Actors System
Secondary Actors User
Precondition System is online, Synchronized button Checked
Main Flow 1. User enter in the app and he/she is already logged in
2. Users go to Settings by pressing on the button “Settings” from the header menu
3. User press on the button that is correspondent to the synchronise button.
4. A loading page appears until the process is done
5. Data will be exactly the same on database and locally
6. Application writes the data on the file from local storage
Post Condition Application synchronise data to database and check if all user data is up to date
Alternative Flows NoInternetConnection, SynchronizedButtonNotChecked
